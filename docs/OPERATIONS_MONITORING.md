# 온집 운영 관찰

> 목적: 운영 배포 후 방문, 설치, 핵심 저장 흐름이 실제로 쓰이는지 가볍게 확인한다.

## 확인 위치

- 운영 앱: `https://onzip.vercel.app`
- Vercel Analytics: 방문 수, 페이지뷰, 기기/브라우저
- GA4: 익명 이벤트, 빠른 추가/저장/설치 흐름
- 이벤트 기준: `docs/ANALYTICS.md`

## 주간 체크

| 항목 | 확인할 것 | 조치 기준 |
|---|---|---|
| 방문 | 방문 수와 재방문 흐름 | 방문은 있는데 저장 이벤트가 없으면 첫 화면/빠른 추가 점검 |
| 기기 | Android/iPhone 비중 | 특정 기기 이탈이 높으면 `docs/DEVICE_QA.md`로 재확인 |
| 설치 | `install_cta_click`, `install_success` | 클릭은 많은데 설치가 적으면 설치 안내 문구 점검 |
| 빠른 추가 | `quick_add_open`, `quick_add_select`, `quick_add_saved` | 열림 대비 저장이 낮으면 입력 폼 단순화 후보 |
| 핵심 저장 | `ledger_saved`, `calendar_event_saved`, `record_saved`, `shopping_saved` | 특정 기능만 낮으면 해당 탭의 빈 상태/버튼명 점검 |
| 백업 | `local_data_export`, `local_data_import` | 교체 이벤트가 늘면 백업 안내와 확인창 재점검 |

## 개인정보 점검

GA4 이벤트를 볼 때 아래 값이 들어가면 즉시 수정한다.

- 제목
- 금액
- 메모 내용
- 태그
- 구성원 이름
- 집 이름
- 사용자가 직접 입력한 자유 텍스트

허용하는 파라미터는 정해진 상태값만이다.

- `type`
- `mode`
- `tab`
- `platform`
- `outcome`
- `status`

## 관찰 로그

```text
기간:
확인자:
방문 흐름:
설치 흐름:
빠른 추가 흐름:
저장 이벤트:
기기/브라우저 특이사항:
개인정보 파라미터 이상 여부:
다음 조치:
```

## 조치 우선순위

| 우선순위 | 기준 | 예시 |
|---|---|---|
| P0 | 개인정보성 이벤트 전송, 앱 실행 불가 | 이벤트 파라미터에 메모 내용이 들어감 |
| P1 | 핵심 저장 흐름 실패 | 빠른 추가 저장 이벤트가 발생하지 않음 |
| P2 | 설치/백업 이해 저하 | 설치 클릭 대비 완료가 지나치게 낮음 |
| P3 | 문구/배치 개선 후보 | 특정 탭 진입은 있지만 저장 전 이탈이 많음 |

## 운영 원칙

- 운영 관찰은 사용자 입력 내용을 보려는 용도가 아니다.
- 숫자가 낮다고 바로 기능을 늘리지 않는다.
- 먼저 `docs/TEST_CHECKLIST.md`, `docs/DEVICE_QA.md`, `docs/USAGE_QA_7D.md`로 재현 가능한 불편인지 확인한다.
- 이벤트를 추가할 때는 `docs/ANALYTICS.md`에 먼저 이름과 의미를 기록한다.
